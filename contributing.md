# Contribution Guidelines

Please note that this project is released with a
[Contributor Code of Conduct](code-of-conduct.md). By participating in this
project you agree to abide by its terms.

---

Ensure your pull request adheres to the following guidelines:

- Suggest resources that are genuinely useful for **technical accounting** —
  authoritative standards, firm guidance, high-signal newsletters, and tools.
- Make sure the resource is freely accessible (or note when registration is
  required) and that the link points to a stable, canonical URL.
- Add one link per pull request so it is easy to review.
- Use the format `[Resource Name](link) - Description.` Keep descriptions
  short and factual, starting with a capital letter and ending with a period.
- Keep the existing alphabetical / topical ordering within each section.
- Check your spelling and grammar.
- New categories or improvements to the existing categorization are welcome.
- Make sure your contribution does not duplicate an existing entry.

Thank you for your suggestions!


## Updating your PR

A lot of times, making a PR adhere to the standards above can be difficult.
If the maintainers notice anything that we'd like changed, we'll ask you to
edit your PR before we merge it. There's no need to open a new PR, just edit
the existing one. If you're not sure how to do that,
[here is a guide](https://github.com/RichardLitt/knowledge/blob/master/github/amending-a-commit-guide.md)
on the different ways you can update your PR so that we can merge it.
